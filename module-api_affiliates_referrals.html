api/affiliates/referrals

is a web interface for module:affiliates/referrals

Methods

(route) bindReferralToAffiliate()

Handlers Chain:
Order
module:middleware/access~addAuthorizationDataToRequest
module:middleware/access~accessUser
this endpoint
Route:
MethodPath
POST /api/affiliates/referrals/bind

it binds the user to the affiliate as a referral. Tariff kind will be read from AffiliateData.selected_tariff_kind_to_bind_referrals. User should not be bound to another affiliate and should have no COMPLETED deposits (see Deposit).

Body Parameters:
Name Type Description
affiliate_id Affiliate.affiliate_id see Affiliate
Response:
Name Type Description
status OK if success
status IMPOSSIBLE for prohibited actions like the attempt to become a referral of yourself
status ALREADY_DONE if user is already bound to any affiliate
status INCORRECT_STATE if user already has completed deposits
status FORBIDDEN if not authorized
status FAIL if internal error

(route) getAffiliateReferralsList()

Handlers Chain:
Order
module:middleware/access~addAuthorizationDataToRequest
module:middleware/access~accessAdmin - required permission scopes: AFFILIATES, SUPPORT
this endpoint
Route:
MethodPath
GET /api/affiliates/referrals/list

It returns list of AffiliateReferral's by .affiliate_id

Query Parameters:
Name Type Attributes Description
affiliate_id Affiliate.affiliate_id
username User.username optional to filter by .username of referral's account (not the Affiliate's username)
user_id User.user_id optional to filter by .user_id of of referral's account
tariff_kind AffiliateTariffKinds optional
offset Number optional default is 0
limit Number optional default is 15
sorting_property String optional default is 'uab.created_at'
sorting_direction String optional 'DESC' or 'ASC', default is 'DESC', new referrals first
Response:
Name Type Description
status OK if success
affiliate_referrals Array.<AffiliateReferral> if success, see AffiliateReferral
status WRONG_ARGS if .affiliate_id not provided
status FORBIDDEN if auth cookie doesnt contain the required permissions scope(s)
status FAIL if internal error
