CrashGamePublicState

Properties:
Name Type Attributes Description
final_multiplier Number <optional>

the multiplier on which the game became "COMPLETED". All players who managed to take the win before the current multiplier has reached this value (inclusive) are the winners. All another players become losers. So, we publish it only when the game has the "COMPLETED" .status
status GameStatus

'CREATED' when the new game is published and is in 'waiting for bets' state with the countdown
token String <optional>

the full JWT token which contains the final multiplier of current game. Can be read using any JWT-decoding service. So, it is presented only when .status is "COMPLETED"
token_signature String

the last part (signature) of CrashGamePublicState.token; For e.g, if the token is "xxx.yyy.zzz", the signature is "zzz". A signature is a kind of salted hash of the token content. We publish the signature once the game is created (and has the "CREATED") status.
game_id BigInt

Unique numeric id of game You should compare it with the .game_id on the client side to prevent mixing the states of different games ("rounds");
started_at String <optional>

the ISODate when game got "IN_PROGRESS" .status; Is not presented for "CREATED" game.
completed_at String <optional>

the ISODate when game got "COMPLETED" .status; Is presented only for "COMPLETED" game.
betting_period_end_timestamp Number <optional>

unix timestamp with millis. Is presented only for "CREATED" game. Means point in time when .status will be updated from "CREATED" to "COMPLETED";
message_sent_at_ms_timestamp Number

unix timestamp with millis. Is the service property to calculate the ping.

This is the schema of messages we broadcast on game status update.
