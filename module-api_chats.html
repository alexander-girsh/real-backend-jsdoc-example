(route) removeChatMessage()

Handlers Chain:
Order
module:middleware/access~addAuthorizationDataToRequest
module:middleware/access~accessAdmin - required permission scopes: SUPPORT
this endpoint
Route:
MethodPath
POST /api/chats/remove-chat-message

It removes the ChatMessage, returns it and publishes the event about that. If parental Chat.chat_type is PUBLIC, all users will receive event about that. Else if it is SUPPORT, the chat owner and admins with SUPPORT permissions scope will receive the event;

Body Parameters:
Name Type Description
chat_message_id ChatMessage.chat_message_id,String
Response:
Name Type Description
status OK if success
removed_message ChatMessage if success, see ChatMessage
status WRONG_ARGS if .chat_message_id not provided
status INCORRECT_STATE if the chat message with provided .chat_message_id and .status='PUBLISHED' does not exist
status FORBIDDEN if auth cookie doesnt contain the required permissions scope(s)
status FAIL if internal error

(route) muteUserInChat()

Handlers Chain:
Order
module:middleware/access~addAuthorizationDataToRequest
module:middleware/access~accessAdmin - required permission scopes: SUPPORT
this endpoint
Route:
MethodPath
POST /api/chats/mute-user-in-chat

it creates a ChatMuting with the provided .until_timestamp_ms. Returns nothing, fires no events. It may affect some unlikely effects when two admins mute one user in same chat in same time: both ChatMuting's will be created. Only two mutings with the exact same .muted_until may not be created

Body Parameters:
Name Type Description
user_id User.user_id,String identifier of user to mute
chat_id Chat.chat_id,String identifier of chat to mute user in. You can mute user only in one chat by one API call.
until_timestamp_ms Number user will have ability to write to the chat again starting from this time
Response:
Name Type Description
status OK if success
status WRONG_ARGS if .user_id .chat_id or .until_timestamp_ms not provided
status FORBIDDEN if auth cookie doesnt contain the required permissions scope(s)
status FAIL if internal error
desc String if not success
